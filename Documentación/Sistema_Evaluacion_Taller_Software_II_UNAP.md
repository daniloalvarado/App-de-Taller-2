SISTEMA DE EVALUACIÓN
El curso se evalúa mediante 6 avances (
sprints
), todos con el 
mismo peso:
Nota Final = (S1 + S2 + S3 + S4 + S5 + S6) / 6
Cada avance corresponde a una iteración funcional del sistema (no se evalúan presentaciones, solo software funcionando).
REGLA CRÍTICA DEL CURSO
La no presentación de uno o más avances impacta dir
ectamente en la nota final del curso. Un 
equipo que no cumpla con las entregas parciales podrá desaprobar el curso, aunque presente un producto final funcional.
EVOLUCIÓN ESPERADA POR SPRINT
Sprint 1: Problema validado + usuario real
Sprint 2
: Prototipo funcional
Sprint 3: MVP funcional
Sprint 4: Funcionalidades completas
Sprint 5: Mejora y estabilidad
Sprint 6: Producto final validado
